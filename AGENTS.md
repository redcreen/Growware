## Current Project Mode

This repository is in experimental runtime foundation mode.

Treat it as a documentation-first project with one approved local experimental runtime path.

Do not treat it as a production-ready software project.

## Current Objective

Growware is defining a local-first closed-loop software evolution system. The current job is to:

- preserve the origin conversation
- evaluate feasibility from the project angle
- keep the first pilot loop and contracts truthful
- implement one local mock runtime that consumes the compiled machine layers without claiming real target-project ownership

## Do Not Do Yet

- do not treat `experiments/mock_runtime/` as proof that `Project 1` runtime wiring is live
- do not widen the experimental runtime into deploy-capable or production-ready behavior by assumption
- do not let experiments silently rewrite the project target, channel binding, approval boundary, or durable policy source
- do not claim autonomous deployment or production readiness
- do not collapse `A window`, `B window`, and the hidden control plane into one undifferentiated chat loop

## Implementation Entry Gate

The paper gate has been recorded explicitly in this repo:

- one concrete pilot target
- one operator path and one real usage path
- one incident contract
- one verification contract
- one deployment approval boundary

Experimental runtime work may now start only under `experiments/mock_runtime/` and must keep deploy and rollback approval-gated.

## Durable Rule

Every human-reported problem that gets resolved should later become a reusable rule, judge, or regression asset once the runtime exists.
