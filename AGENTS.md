## Do Not Do Yet

- do not guess the runtime, framework, or packaging model
- do not scaffold app code just to make the repo feel concrete

- do not collapse `A window`, `B window`, and the hidden control plane into one undifferentiated chat loop

## Implementation Entry Gate

Implementation can start only after the repo explicitly records:

- one concrete pilot target
- one operator path and one real usage path
- one incident contract
- one verification contract
- one deployment approval boundary

## Durable Rule

Every human-reported problem that gets resolved should later become a reusable rule, judge, or regression asset once the runtime exists.
