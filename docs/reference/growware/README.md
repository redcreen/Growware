# Growware Reference Pack

[English](README.md) | [中文](README.zh-CN.md)

## Purpose

This directory holds Growware's durable reference sources. It is not the live control surface; it is the long-lived layer for project definition, origin material, and stage-level judgments.

## Recommended Reading Order

- [origin.pdf](origin.pdf): the full original conversation, now the canonical source
- [origin-raw-extract.zh-CN.md](origin-raw-extract.zh-CN.md): the full Markdown archive extracted directly from the PDF for manual review
- [origin.md](origin.md): the extracted project definition from the full origin conversation
- [origin-transcript-2026-04-13.md](origin-transcript-2026-04-13.md): the share-page transcript capture, retained only as provenance and known to be incomplete
- [feasibility.md](feasibility.md): project-level feasibility, risks, and entry conditions
- [pilot-loop-v1.md](pilot-loop-v1.md): the first pilot loop definition and implementation gate for Project 1
- [daemon-foundation-plan.md](daemon-foundation-plan.md): the current daemon-first planning line for improving Growware itself before broadening target-project execution
- [daemon-contracts/README.md](daemon-contracts/README.md): the durable daemon contract pack that turns the planning line into machine-compilable source docs
- [stage-2-3-baseline.md](stage-2-3-baseline.md): the Stage 2 and Stage 3 paper baseline that closes the delivery, gate, judge, and automation rules without claiming runtime completion
- [stage-2-3-contracts/README.md](stage-2-3-contracts/README.md): the detailed source pack for incident, verification, deploy gate, provenance, automation bands, and regression assets
- [shared-policy-contract.md](shared-policy-contract.md): the shared policy contract for humans, `project-assistant`, and Growware, covering the current pilot `.growware/*` compatibility profile, the long-term `docs/policy/` -> `.policy/` target, compile responsibility, execution boundaries, and layered gates
- [../../policy/README.md](../../policy/README.md): the human-readable policy source entry for Project 1
- [development-plan.md](development-plan.md): detailed execution queue below the roadmap

## Reading Rules

- treat `origin.pdf` and `origin*.md` as the source of truth for what the project is
- treat `feasibility*.md` as the source of truth for whether the current direction is realistic
- treat `architecture*.md`, `roadmap*.md`, `development-plan*.md`, `pilot-loop-v1*.md`, `daemon-foundation-plan*.md`, `daemon-contracts/*`, `stage-2-3-baseline*.md`, and `stage-2-3-contracts/*` as the source of truth for the current recommended shape, implementation gate, and current paper-complete order of work
